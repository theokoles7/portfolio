Quizzler

GitHub Live Demo

Description

Quizzler was born from the interest my friends and I have in trivia — we can be a bit obsessive about it. We had played certain trivia games so many times that we were running out of fresh questions, which gave me the idea to simply create my own trivia app. I made it a point to keep it simple, clean, and easy to use.

When arriving at the main menu, users will find multiple options for customizing their round of questions: quantity, category, difficulty, and type (multiple choice or true/false) can all be controlled. Once the player selects "Begin", they are taken to a scrolling question stream and can start answering. Fair warning: even questions labeled "Easy" can be quite difficult. Personally, I have yet to get a perfect score.

Lessons Learned

At the surface, this project may not seem very significant, but it is important to me because it entailed a lot of firsts. It was the first app I ever made from beginning to end by myself. It was the first app I deployed publicly. It was the first app I implemented a controlled theme for — I actually created two themes that users can switch between. It was the app that taught me the largest amount of HTML and CSS I had ever learned all at once, and the first time I experimented with mobile responsiveness.

Since creating Quizzler, I have been able to carry all of these lessons over to new projects, and I believe I have become far more efficient. Where it took me around a month to create Quizzler, it now takes me a few days to create more complicated apps.

Tools Used

Built using Angular with HTML, SCSS, and TypeScript. The only styling library used is Bootstrap Icons. In order to deploy without a database of preloaded questions I would have knowledge of, I utilized a free API called Open Trivia Database. GitHub Pages is used for deployment.
